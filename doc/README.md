# 二次开发推进文档索引

> 适用项目：`maplibre-gl-js` 二次开发
>
> 更新时间：2026-02-12

## 1. 文档目标

本目录用于沉淀二次开发方案，保证：

- 架构调整有统一蓝图；
- 需求扩展有接口契约；
- 技术债治理有优先级；
- 性能优化有量化指标；
- 兼容性与发布有可执行流程。

## 2. 文档清单

| 文档 | 作用 | 主要读者 | 产出物 |
| --- | --- | --- | --- |
| `01-现状评估与目标.md` | 明确现状、问题与目标边界 | 架构师/技术负责人 | 二开范围与成功指标 |
| `02-目标架构与模块边界.md` | 定义目标架构、模块职责、依赖规则 | 架构师/核心开发 | 模块拆分与重构约束 |
| `03-接口定义与扩展规范.md` | 统一接口契约与扩展机制 | 平台开发/业务开发 | API/事件/扩展规范 |
| `04-技术债治理计划.md` | 技术债分类、优先级与治理节奏 | 全体开发 | 可执行治理 Backlog |
| `05-性能优化专项方案.md` | 性能指标、分析方法、优化路线 | 图形/引擎开发 | 性能专项计划与验收口径 |
| `06-向后兼容与发布策略.md` | 版本策略、兼容矩阵、灰度方案 | 架构师/发布负责人 | 兼容与发布流程 |
| `07-开发路线图与里程碑.md` | 分阶段推进计划与资源配置 | 项目经理/技术负责人 | 12~16 周路线图 |
| `08-风险清单与应对.md` | 风险识别、监控与应急预案 | 管理层/技术负责人 | 风险台账与预案 |
| `09-执行模板与验收标准.md` | 周报、提案、评审与验收模板 | 全体开发 | 标准化执行模板 |

## 3. 推荐使用方式

- 启动阶段：先阅读 `01` + `07`，确认目标与排期。
- 设计阶段：按 `02` + `03` 输出架构图和接口评审稿。
- 实施阶段：按 `04` + `05` 同步推进治理和性能专项。
- 发布阶段：按 `06` 执行兼容性验证与灰度发布。
- 周期复盘：按 `08` + `09` 更新风险台账和执行模板。

## 4. 维护约定

- 每次里程碑结束后更新相关文档“变更记录”段落；
- 接口或模块边界变更必须同步 `02` 与 `03`；
- 发布流程改动必须同步 `06` 与 `09`；
- 变更建议通过 PR 方式提交，确保可审计。
